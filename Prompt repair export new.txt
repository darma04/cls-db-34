Saya memiliki sebuah Web Application (berbasis Django + jQuery DataTables) untuk Manajemen Kos. Saat ini fitur Export Excel dan PDF pada tabel-tabel data mengalami banyak bug, terutama ketika pengguna mencoba menyembunyikan/mem-filter kolom.

Lakukan audit dan perbaikan menyeluruh pada seluruh halaman yang memiliki fitur tabel dengan DataTables dan Export. Terapkan 4 Langkah Standarisasi berikut secara ketat dan jangan lewati satu pun:

Langkah 1: Perbaiki Bug "Crash Setelah Filter Kolom" pada Export Akar masalah: Fungsi 

exportToExcel()
 dan 

exportToPDF()
 lama menggunakan seleksi DOM statis (contoh: cells[5]). Jika kolom disembunyikan menggunakan filter 

toggleColumns()
, indeks kolom bergeser dan skrip akan crash. Instruksi:

Rombak fungsi export. Gunakan DataTables API untuk mengambil hanya kolom yang sedang visible.
Lakukan iterasi dengan tableObj.columns().every(function(index) { ... if (this.visible() && headerText !== 'aksi') { visibleColumns.push(...) } }) untuk mendata kolom yang aktif.
Ekstrak data setiap baris dengan tableObj.cell(rowIdx, col.index).node().textContent.
Langkah 2: Perbaiki Bug "DataTable Re-initialization Crash" Akar masalah: Fungsi filter kolom (biasanya bernama 

toggleColumns()
, 

applyFilter()
, atau aksi pada checkbox) memanggil $('#tableId').DataTable() yang menyebabkan DataTables melakukan re-initialization di luar $(document).ready dan menghancurkan instance tabel. Instruksi:

Cari (Grep) ke SELURUH file HTML. Semua pemanggilan $('#tableId').DataTable() di dalam blok fungsi 

toggleColumns()
 WAJIB diganti menjadi metode safe retrieval yaitu: new $.fn.dataTable.Api('#tableId').
Langkah 3: Integrasikan Template Cetak (Kop Surat & Footer) Akar masalah: Export bawaan tidak menyertakan identitas perusahaan. Instruksi:

Masukkan variabel template backend (seperti nama perusahaan, alamat, telepon, dan copyright) ke dalam variabel JS konstan (EXCEL_INFO dan PDF_INFO) sebelum fungsi export.
Tambahkan properti ini di area atas dokumen Excel (sebagai baris tambahan <tr>) dan area Header/Footer pada konfigurasi pdfMake dengan posisi landscape dan margin yang proporsional.
Langkah 4: Perbaiki Bug Baris Ringkasan (Tfoot) Ekspor Kosong / Tertukar Akar masalah: Fungsi ekspor lama menggunakan DOM Traversal (document.querySelector('tfoot').cloneNode) untuk mengambil nilai ringkasan (Total Pemasukan, dll). Cara ini rentan rusak akibat kode HTML kompleks (seperti elemen <span>, <badge>, atau <i>) dan pergeseran colspan. Instruksi:

JANGAN menggunakan teknik cloneNode untuk tfoot saat mengekspor.
Alih-alih merayapi DOM, buat konfigurasi Hardcoded Data Mapping di JavaScript (contoh: const FOOTER_DATA = { 4: "{{ total_pemasukan|escapejs }}", 5: "{{ total_pajak }}" };).
Saat skrip mengekspor dan mencapai baris tfoot, interogasi array visibleColumns. Jika indeks kolom tersebut ada di dalam objek FOOTER_DATA, cetak nilainya pada cell dokumen Excel/PDF. Jika indeks kolom tidak ada, hitung jumlah kolom berturut-turut yang tidak memiliki nilai (sebagai jarak spacer) lalu cetak gabungannya menggunakan <td colspan="n">.
Khusus untuk jarak spacer pertama (paling kiri), cetak teks "RINGKASAN".
Terapkan 4 Langkah Standarisasi ini dengan menulis Script Python atau multi-replace yang meng-otomatisasi pergantian kode secara massal ke semua menu Manajemen Kos (seperti Daftar Kamar, Penyewa, Tagihan, Laporan Pembayaran, Pengeluaran, dll). Pastikan HTML tidak memiliki syntax error setelah injeksi kode selesai. Uji keberhasilan export secara teliti!
